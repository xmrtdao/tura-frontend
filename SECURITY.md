# Security Policy

## Supported Versions

We actively support the latest version of each repository. Security patches are applied to the main branch.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT open a public issue** — this could expose the vulnerability to malicious actors.
2. Email details to: security@xmrtdao.org
3. Include: description, reproduction steps, potential impact, and suggested fix (if any).
4. Allow up to 72 hours for an initial response.

## Disclosure Timeline

- **Day 0**: Vulnerability reported
- **Day 3**: Initial response and triage
- **Day 14**: Patch developed and tested
- **Day 30**: Coordinated disclosure (if applicable)

## Security Best Practices for Contributors

- Never commit API keys, passwords, or private keys.
- Use environment variables for sensitive configuration.
- Keep dependencies up to date.
- Follow the principle of least privilege.

## Acknowledgments

We appreciate responsible disclosure and will acknowledge reporters in our release notes (with their permission).
